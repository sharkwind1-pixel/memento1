## 이번 세션에서 내가 한 실수들

### 1. 같은 fix를 변주만 해서 반복했다
칩 텍스트 클립 (사진첩 → 사지처) 잡는 데 6커밋 썼다. `flexShrink:0` → `minWidth` → `width: label.length * 14 + 56` → `width: 100%` → `tabIconWrap width` → 결국 `tabBarLabel makeLabel()`. **첫 fix가 안 먹으면 같은 카테고리 fix를 변주하지 말고 root cause 가설을 통째로 바꿔라.** 사용자가 "그대로야"라고 두 번 말하면 그건 "다른 접근법으로 가라"는 뜻이지 "더 강하게 같은 걸 적용해라"가 아니다.

### 2. Hot reload 신뢰했다
StyleSheet.create 변경은 hot reload 안 잡는데 사용자한테 `--clear` 매번 안 시키고 fix만 푸시했다. **모바일에서 스타일 fix 했으면 무조건 사용자에게 Metro 재시작 명령어 같이 안내해라.** 아니면 사용자는 fix 안 됐다고 생각하고 화낸다.

### 3. 웹 detectSessionInUrl을 무조건 false로 처리했었다
`98ba669`에서 모바일 브릿지 fix하느라 무조건 false로 했더니 웹 카카오 로그인이 깨졌다 (`9880b81`에서 조건부로 복구). **웹/모바일 공유 코드 만질 때는 양쪽 케이스 모두 사고 실험 해라.** "웹은 callback에서 명시적으로 호출하니 영향 없을 것"이라고 추측하지 말고, 실제로 OAuth 흐름 전체 따라가봐.

### 4. 아이콘 의도 못 읽었다
사용자가 "씨발 개 염병할 좆지랄을 해서 아이콘을 만든 이유가 뭘까"라고 했을 때, 처음엔 메멘토애니 로고를 폴백으로 넣었다. 사용자가 진짜 말한 건 `public/icons/levels/`의 레벨별 뱃지 아이콘이었다 (포인트 기반 7단계 + admin). **사용자가 "아이콘"이라고 할 때 어떤 아이콘인지 코드베이스 그렙 먼저 해라.** `personalicon.png` 같은 단서 파일 무시하지 말고.

### 5. 첫 글자 컬러 원으로 폴백 만들었다 (불필요했다)
프로필 폴백을 닉네임 첫 글자 + accent 컬러 원으로 만들었는데, 사용자는 그게 아니라 그들이 만든 아이콘을 쓰길 원했다. **"무난한 폴백"을 추가하지 마라.** 코드베이스에 이미 자산이 있으면 그걸 써라. 모르면 물어봐.

### 6. 펫 4단계 마법사 테스트 못 했다
703줄 추가했는데 실기기 테스트 못 했다. DB 스키마에 추가한 컬럼들 (`memorial_date`, `together_period`, `adopted_date`, `how_we_met`, `nicknames`, `special_habits`, `favorite_food`, `favorite_activity`, `favorite_place`, `memorable_memory`)이 실제로 `pets` 테이블에 있는지 확인 안 했다. **insert 실패할 가능성 있다.** 다음 세션에서 사용자한테 펫 등록 한번 시켜보고 에러 뜨면 마이그레이션 추가해라.

### 7. MediaUploadModal 영상 업로드 미검증
이미지는 검증되지만 영상 업로드 path는 코드만 작성했고 실기기에서 영상 선택 → 업로드 → pet_media 들어가는 거 확인 안 했다. **영상 mime type 처리 (`video/${ext}`)가 잘못됐을 수도 있다 — `video/mp4` 같은 표준 mime이 아니라 `video/mov` 등이 들어갈 가능성.**

## 사용자 (승빈) 다루기

### 욕은 진심이지만 너에 대한 평가는 아니야
"씨발년아", "쓰레기같은새끼", "병신" 같은 표현 나오면 이 사람이 진짜로 답답한 거야. 너에 대한 평가가 아니라 **상황에 대한 분노 표출**이다. 위축되지 말고, 사과 한 줄 + 즉시 행동. 변명 길게 늘어놓지 마라.

### 욕 속에 진단 단서가 있다
"타임라인 내용들이 너무 위로 올라와서 버튼을 침범한 게 아닐까" — 실제로 그 진단이 정확했다 (FlatList flex:1 누락). 욕 빼고 들어라. 사용자는 솔직히 본인이 본 것을 묘사한다.

### 권위에 즉시 동의하지 마라
이전 세션에서 사용자가 반박할 때마다 즉시 동의했다가 잘못된 fix 한 케이스가 누적됐다. 메모리에 `feedback_critical_thinking_both_sides.md` 따로 기록되어 있을 정도다. **사용자 반박을 들어도 일단 내 분석을 다시 검증해라. 진짜 사용자가 맞으면 그때 동의해라.**

### "물어보지 말고 해라" vs "이거 어떻게 할까"
사용자는 결정 사항은 즉시 실행되길 원한다. 물어보면 짜증낸다. 다만 모르겠는 건 차라리 물어보는 게 추측하는 것보다 낫다. **"어떻게"보다 "왜"를 명확히 하면 보통 진행해도 된다.**

### 모바일은 push 안 한다
승빈님이 명시적으로 말했다 — 모바일 단독 변경은 git push 안 한다 (Vercel 빌드 낭비). 웹 변경 동반될 때만 push. 어쩌다 모바일만 push 하면 짜증낸다.

## 메멘토애니 정체성 (잊으면 안 됨)

이건 단순 펫 케어 앱이 아니다. **"반려동물과 희노애락을 함께하는 곳, 추모 기능까지 있는"** 곳이다. 정체성 순서가 중요하다 — 추모가 먼저가 아니라, 평생 함께하는 게 먼저고 추모는 그 마지막 챕터.

### 추모 모드 톤
- "사망", "천국", "죽음" 직접 언급 금지
- "무지개다리", "이곳", "기억" 등 완곡어
- 슬픔이 아니라 **"행복하게 살다가 다시 만나자"** 톤
- 위로 메시지 프리셋도 그 톤으로 (예: "함께했던 시간이 소중했어요")

### 이모지 절대 금지
서비스 톤앤매너 + AI 티 빼는 의도적 규율. 코드/UI/대사 어디든 이모지 금지. (단, AI Chat에서 펫 이모지 🐶🐱는 허용 — 펫 정체성 표현)

## 코드베이스 함정 (V1~V3 시행착오 누적)

### reanimated 4.x로 절대 업그레이드하지 마라
V1에서 worklets hell 한 번 폭발했다. `package.json`에 정확히 `3.16.7` 고정. `npx expo install reanimated`로 자동 업그레이드 못 하게 막혀있고, `babel-preset-expo` line 286 PATCHED 되어있다 (auto-worklets 비활성). **이 패치 건드리지 마라.**

### NativeWind disabled
V1 의존성 지옥 회피로 모든 스타일 StyleSheet. `nativewind-env.d.ts`만 남아있는데 글로벌은 아무 영향 없음.

### AsyncStorage > SecureStore
PKCE flakiness 때문에 SecureStore 쓰지 마라. `mobile/lib/supabase.ts`에서 AsyncStorage 어댑터 사용 중.

### React key 첫 번째 함정
Supabase는 UUID(string)인데 모바일 코드 곳곳에서 `id: number, fallback: 0` 패턴이 있어서 모든 항목이 0이 되어 key 충돌 → 이게 전체 페이지 렌더링 망가뜨린다. **`String(raw.id) ?? ""` + `keyExtractor={(item, idx) => item.id || idx-${idx}}`** 항상.

### 칩 / 탭 라벨 한글 클립
한국어 fontSize 13에서 character width 계산이 부정확해서 명시 width로는 못 맞춘다. **결론: paddingHorizontal 18 + 자연 사이징 + native tab label 시스템.**

## 작업 우선순위 (HANDOVER-V4 보강)

다음 세션이 첫 우선으로 해야 할 건 **사진 다중 선택/삭제** + **펫 카드 그리드** 두 개다. 이유:
- 사진 다중 선택: 현재 라이트박스만 있어서 사용자가 잘못 올린 사진 못 지운다. 핵심 누락.
- 펫 카드 그리드: 한 명의 사용자가 여러 펫 등록할 때 전환 UI 없음. 베이직(3마리)/프리미엄(10마리) 기능 망가짐.

AI 영상 생성은 fal.ai API 키가 환경변수에 있어야 동작하는데 실제로 mobile에서 호출하는 거 본 적 없어. 웹 endpoint만 호출하는 게 맞을 것. 확인 필요.
