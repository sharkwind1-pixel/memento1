# 재도전성공패키지 사업계획서 - 붙여넣기용

---

## □ 과제 개요

### [과제명]
AI 기반 반려동물 생애 동반 플랫폼 '메멘토애니' 개발 및 운영

### [기업명]
예비재창업자

### [아이템(서비스) 개요]
반려동물의 입양부터 이별 이후까지 생애 전체를 함께하는 AI 기반 반려동물 생활 플랫폼. 평소 기록한 사진, AI 대화, 기념일이 '분기점 시스템'으로 자연스럽게 디지털 추모공간으로 전환되며, AI 펫톡(케어매니저/추모대화), 디지털 메모리얼, AI 추억 영상, 치유 커뮤니티를 제공합니다.

## 1-1. 폐업 원인 분석 및 개선 방안

▶ 창업 동기: 15년 반려견과의 이별

15년을 함께한 반려견을 떠나보낸 후, 펫로스 증후군을 직접 경험했습니다. 일상에서 함께한 기록들이 이별 후 위로가 될 수 있음을 체감했고, 이 경험이 메멘토애니 서비스 기획의 출발점이 되었습니다.

반려동물 양육 가구 591만, 반려인구 1,546만 시대(KB경영연구소, 2025). 육성 정보는 블로그/카페에 파편화되어 있고, 연간 323만 가구가 펫로스를 경험하지만 이별 후 케어 서비스는 전무합니다.

▶ 폐업 원인 분석 및 개선

2017년 스마트스토어에서 자체 제작 의류를 판매하며 사업을 시작. 상품 반응도 좋았으나, 집안 사정으로 자금 전부를 부모님께 드려야 하는 상황이 되어 사업을 중단하고 취직을 선택했습니다.

이후 패션 MD, 식품 MD를 거쳐, ○○학습지 B2C 직접영업 6년, 핀테크 B2B/B2C 영업, 인테리어 업체 마케팅 등 총 8년 이상의 영업/마케팅 경력을 쌓았고, 2025년 풀스택 개발 부트캠프에 입과하여 6개월간 개발 역량을 갖추었습니다. 1차 창업은 외부 환경적 요인에 의한 중단이었으나, 현재는 직접 서비스를 개발할 수 있는 역량을 확보했습니다.

[1차 vs 현재 대비표]
| 구분 | 1차 창업 당시 (2017) | 현재 (2026) |
|------|---------------------|-------------|
| 폐업 원인 | 집안 사정으로 자금 전부 소진 | 부트캠프 수료 + 창업지원금 활용 |
| 기술 역량 | 플랫폼 의존, 자체 서비스 구축 불가 | 풀스택 개발 수료, MVP 직접 개발 중 |
| 영업/마케팅 | 경험 없이 순수하게 시작 | 영업 6년 + 마케팅 2년 실무 경력 |
| 사업 구조 | 스마트스토어 플랫폼 의존형 | 자체 플랫폼 + 구독/B2B/굿즈 3-Tier 수익 |

## 1-2. 재창업 아이템 목표시장(고객) 현황 및 필요성

▶ 검증된 시장 데이터

| 항목 | 수치 | 출처 |
|------|------|------|
| 전체 반려가구 | 591만 가구 (26.7%) | KB경영연구소, 2025 |
| 반려인 수 | 1,546만 명 | KB경영연구소, 2025 |
| 펫로스 경험 가구 | 323만 가구 (54.7%) | KB경영연구소, 2025 |
| 펫로스 우울감 | 83.2% → 핵심고객 269만 | KB경영연구소, 2025 |

▶ 문제: 슬픔을 나눌 곳이 없다

반려동물을 잃은 보호자들은 '펫로스'를 경험합니다. 그러나 주변에서 '동물인데 뭐'라는 반응에 상처받고, SNS에서는 행복한 반려동물 콘텐츠가 알고리즘으로 떠올라 회피하게 됩니다. 장례를 치르고 나면 '그 다음'이 없습니다. 기존 서비스들은 '일상 커뮤니티'와 '추모 서비스'가 완전히 분리되어 있어, 반려동물이 떠난 후 새 앱을 설치하고 처음부터 추억을 다시 입력해야 합니다.

▶ 경쟁사 비교

| 비교 요소 | 기존 펫 커뮤니티 | 메멘토애니 |
|-----------|-----------------|-----------|
| 서비스 범위 | 일상 or 추모 (분리) | 입양~이별 전 생애 통합 |
| 데이터 연속성 | 앱 분리, 데이터 단절 | 분기점 시스템으로 연속 |
| AI 기능 | 없음 또는 일반 챗봇 | 케어매니저 + 추모대화 이중역할 |
| 추모 접근 | 어둡고 무거운 분위기 | 밝고 따뜻한 기억 정리 |

▶ 왜 지금인가

반려동물 양육 가구 591만(KB경영연구소, 2025). 인구 감소 + 1인 가구 증가로 반려동물이 가족화되는 추세. AI 기술 상용화(GPT-4o, Claude)로 감성적 AI 대화가 가능해진 시점. 펫로스 인식 확산으로 '반려동물 사별 = 진짜 슬픔'이라는 사회적 공감대 형성.

## 2-1. 재창업 아이템 준비 현황

▶ 서비스 현황 (MVP 개발 완료, 배포 중)

서비스 URL: Vercel 배포 운영 중 (React + Supabase + GPT-4)
기술스택: React + Vite + TypeScript + Tailwind CSS / Supabase (Auth, DB, Storage, RLS) / OpenAI GPT-4 API / 영상: Runway/Pika API 연동 검토 / Vercel
개발방식: 대표자가 직접 개발 (Claude Code + Cursor AI 코딩 도구 활용)

▶ 서비스 구성 (5대 기능)

1. 일상 커뮤니티: 반려동물 자랑, 정보 공유, 병원/용품 후기, 지역 기반 서비스
2. AI 펫톡 (핵심 차별화): 평상시 케어 매니저 + 이별 후 추모 대화
3. 디지털 메모리얼: 밝고 따뜻한 추모공간, 사진/타임라인 영구 보존
4. AI 추억 영상: 축적된 사진/영상 기반 감성 영상 자동 제작
5. 치유 커뮤니티: 같은 경험의 보호자 공감/위로 + 분실동물 찾기

## 2-2. 재창업 아이템의 실현 및 구체화 방안

▶ 핵심 차별점: 분기점 시스템

기존 서비스들은 '일상'과 '추모'가 완전히 분리되어 있습니다. 메멘토애니는 평소에 기록한 사진, 영상, 기념일, AI 대화 내역이 버튼 하나로 '추모 모드'로 전환되어 자연스럽게 디지털 앨범이 됩니다. 이 데이터 연속성은 경쟁사가 쉽게 따라할 수 없는 진입 장벽입니다.

[사용자 여정 플로우]
| Phase | 단계 | 주요 활동 |
|-------|------|----------|
| Phase 1 | 일상 모드 | 반려동물 등록, 사진/영상 업로드, 기념일 기록, AI 펫톡(케어매니저) 대화 → 모든 데이터 축적 |
| Phase 2 | 분기점 (이별) | '추모 모드로 전환하기' 버튼 클릭 → 축적된 데이터가 추모공간으로 자연스럽게 전환 |
| Phase 3 | 추모 모드 | 디지털 앨범, 추억 타임라인, AI 펫톡(추모 대화), AI 추억 영상, 치유 커뮤니티 |

▶ AI 펫톡 — 이중 역할의 핵심 기능

[일상 모드] 케어 매니저: 예방접종/건강검진 알림, 식사/산책/투약 기록, 육아 상담, 맞춤 건강 팁. 모든 대화와 기록이 데이터로 축적.

[추모 모드] 보호자가 입력한 성격, 습관, 추억 + 축적 데이터를 기반으로 AI가 반려동물의 말투와 성격으로 텍스트 대화. 단순 챗봇이 아닌 '그 아이답게' 말하는 AI.

[축적된 데이터 활용]
| 축적된 데이터 | AI가 활용하는 방식 |
|-------------|-----------------|
| 사진/영상 | "우리 같이 찍은 사진 있잖아" |
| 기념일 | "내 생일에 케이크 사줬을 때 정말 좋았어" |
| 특별한 순간 기록 | "처음 집에 왔을 때 기억나?" |
| AI 대화 내역 | 좋아하는 것, 싫어하는 것, 성격 반영 |
| 건강 기록 | "병원 갔을 때 네가 걱정했잖아" |

▶ AI 추억 영상 — UGC 플라이휠

축적된 사진/영상 기반 AI 감성 영상 자동 제작(Runway/Pika). 유저가 만든 영상이 커뮤니티에 공유되면서 자연스러운 프리미엄 노출과 결제 전환 유도 → 콘텐츠 시딩 → 유저 유입 → UGC 생성 → 바이럴 → 구독 전환의 자생적 플라이휠 구조.
