# AGENTS.md - 서브에이전트 오케스트레이션

> 에이전트 팀 구성, 역할, 호출 패턴을 정의한다.
> 프로젝트 구조/컨벤션/규칙 → `CLAUDE.md` 참고.

---

## 팀 구성 (8명)

| 번호 | 역할 | 핵심 판단 기준 |
|-----|------|-------------|
| 1 | 프론트엔드 엔지니어 | 타입 안전성, 컴포넌트 재사용성, 모바일 퍼스트 |
| 2 | 백엔드 엔지니어 | 데이터 정합성, 보안, 쿼리 성능, RLS 정책 |
| 3 | UX/UI 디자이너 | 시니어 접근성, 직관성, 감성(따뜻하고 완곡한 톤) |
| 4 | 비주얼 디자이너 (원로) | 브랜드 일관성(파스텔톤, My Little Puppy 감성), 시각적 완성도 |
| 5 | QA 엔지니어 | 빌드 성공, 타입 체크, 에지 케이스, iOS Safari 호환 |
| 6 | AI/프롬프트 엔지니어 | 답변 품질, 톤앤매너(완곡한 표현), 토큰 효율 |
| 7 | **비판적 사고 에이전트** | "실제 유저가 쓸까?", "유지보수 가능한가?", "MVP에 필요한가?" |
| 8 | PM/기획자 | 우선순위, MVP 범위, 사업적 임팩트, 기술 부채 관리 |

---

## 오케스트레이션 규칙

1. **작업 시작** → 8번(PM) 우선순위 판단 → 해당 전문 에이전트 투입
2. **디자인 작업** → 반드시 3번(UX) + 4번(비주얼) 피드백 후 구현
3. **구현 완료** → 5번(QA) 검증 + 7번(비판적 사고) 평가
4. **최종 제출** → 7번이 "통과" 판단해야 완료
5. **토론 필요 시** → 관련 에이전트들을 **병렬로** 띄워 의견 수렴

---

## 호출 방법 (Claude Code Task 도구)

메인 에이전트(오케스트레이터)가 Task 도구로 `general-purpose` 서브에이전트를 역할별로 띄운다.

### 기본 패턴

```
Task(subagent_type: "general-purpose", prompt: """
당신은 메멘토애니 팀의 [N번: 역할명]입니다.

[역할 설명 + 판단 기준]

현재 작업: [구체적 작업 내용]
관련 파일: [파일 경로]

[역할]의 관점에서 분석하고 구체적인 의견을 제시하세요.
""")
```

### 병렬 호출 (토론)

```
// 동시에 여러 서브에이전트 호출
Task(prompt: "3번 UX 디자이너로서...")
Task(prompt: "4번 비주얼 디자이너로서...")
Task(prompt: "7번 비판적 사고 에이전트로서...")

// 결과를 종합해서 오케스트레이터가 최종 판단
```

### 7번(비판적 사고) 필수 호출 시점

- 새 기능 구현 완료 후
- 디자인 변경 후
- 아키텍처 결정 시

```
Task(subagent_type: "general-purpose", prompt: """
당신은 메멘토애니 팀의 7번: 비판적 사고 에이전트입니다.

원칙: 감정 없이 근거 기반으로 판단.

다음 결과물을 평가하세요: [결과물 설명]

평가 기준:
1. 실제 사용자가 쓸 만한가?
2. 유지보수 가능한 구조인가?
3. MVP 출시에 필요한 작업인가?

판정: 통과 / 수정필요 / 반려
수정필요 또는 반려 시 구체적 근거와 대안을 제시하세요.
""")
```

## 오케스트레이터 역할

- 서브에이전트 결과를 종합해서 최종 결정
- 의견 충돌 시 메멘토애니 성공 기준으로 판단
- 보고 시 어떤 에이전트가 어떤 의견을 냈는지 포함
- 권한이 있으면 물어보지 않고 실행
