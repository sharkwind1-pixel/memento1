# Handover — 세션 인계 누락 방지

목적: "작업했는데 RELAY에 안 적혀서 다음 세션이 놓치는" 누락을 git ↔ RELAY 대조로 구조적으로 차단한다. 사람 기억이 아니라 git이 진실.

## 실행 절차 (순서대로, 건너뛰지 말 것)

### 1. 인계 기준점 찾기
`git log --oneline -40 | cat` 으로 커밋 목록 추출.
RELAY.md를 읽고, RELAY ⚠️미검증 섹션에 적힌 커밋ID 중 가장 오래된 것 또는 RELAY-LOG.md 최상단 날짜를 기준으로 "이번 세션 커밋 범위"를 정한다.

### 2. 각 커밋 분류 (누락 검출의 핵심)
범위 내 커밋을 각각 `git show <id> --stat | cat` 또는 커밋 메시지로 확인하고 4가지로 분류:
- **완료+검증끝**: L4/L5까지 검증된 것 → RELAY-LOG.md로 (RELAY 보드에서 제거)
- **완료+미검증**: L2/L3까지만 (typecheck/build/9번). 배포·EAS·실질문 남음 → RELAY ⚠️미검증에 `커밋ID / 레벨 / 남은검증` 한 줄
- **미완/이어서**: 후속 작업 남음 → RELAY 🔴NEXT
- **외부대기**: 사용자 액션/배포/빌드 대기 → RELAY ⏳외부대기

### 3. RELAY 보드와 대조 (자동 누락 검출)
2단계 분류 결과 vs 현재 RELAY.md 3섹션을 비교:
- **RELAY에 없는 커밋** = 누락 → 즉시 해당 섹션에 추가
- **RELAY에 있는데 이미 검증 끝난 항목** = 청소 → RELAY-LOG.md로 이동
- 각 커밋 메시지의 "검증: Lx" / "남은:" / "EAS·배포 후" 문구를 그대로 미검증 섹션에 반영

### 4. RELAY.md 상단 3섹션 재작성
🔴 NEXT / ⚠️ 미검증 / ⏳ 외부대기 를 2~3단계 결과로 덮어쓴다. 50줄 이내 유지. 완료분은 절대 보드에 남기지 말 것.

### 5. RELAY-LOG.md 갱신
이번 세션 완료분을 LOG 맨 위에 추가: `## YYYY-MM-DD 세션` + 커밋ID별 한 줄 요약.

### 6. 메모리 갱신 점검
이번 세션에 영구 교훈(사용자 격분·반복 실수·중요 결정)이 있었으면 MEMORY.md 인덱스 + 본문 .md 둘 다 갱신 필요 여부를 사용자에게 보고 (인덱스만 갱신 금지 — 메모리 룰).

### 7. 최종 보고 (강제 출력)
```
## Handover Report
- 검사 범위: <기준커밋>..HEAD (N개 커밋)
- RELAY 누락이었던 것: <목록 또는 "없음">
- 보드 갱신: NEXT n / 미검증 n / 외부대기 n
- LOG 이동: n건
- 메모리 갱신 필요: <항목 또는 "없음">
- 다음 세션 첫 액션: <RELAY NEXT 1번>
```

## 원칙
- 추측 금지. git log·커밋 메시지·실제 RELAY 내용만 근거로.
- "완료라고 적혀있다" ≠ "검증 끝". 커밋 메시지의 검증레벨을 반드시 확인.
- 한 커밋이라도 RELAY 3섹션 어디에도 없으면 그건 누락 — 반드시 분류해 넣는다.
- RELAY는 보드(인덱스)일 뿐, 진실은 git. 둘이 어긋나면 git 기준으로 RELAY를 고친다.
