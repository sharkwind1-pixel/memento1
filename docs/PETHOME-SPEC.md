# 펫홈(Pet Home) 기능명세 · 화면설계서

> 상태: **초안 v0.1 (논의·합의용, 구현 전)** · 작성 2026-06-09
> 목적: 회원가입 전환이 막히는 문제를, "싸이월드 미니홈피 감성의 펫홈"을 와꾸로 삼아
> 풀기 위한 정보구조 · 화면 · 기능 · 전환후크 · 일상↔추모 뷰 · 바이럴 루프 · 패리티를 한 장에 정리.
> 이 문서로 합의 후 단계별 구현. **아직 코드 변경 없음.**

---

## 0. 배경 / 문제정의

- 가입 *버튼*(소셜 원탭, 카카오/구글/네이버)은 마찰이 낮음. 진짜 누수는 **그 앞단의 동기**.
- 현재 비회원 첫 화면(`HeroSection` 비로그인)이 "특별한 매일을 함께 / 기록·간직"이라 *일기앱* 프레이밍 → 저관여·"나중에".
- 강력한 후크(AI펫톡·AI영상)는 벽 뒤 + 탭 구석이라 *맛보기 전에 가입 요구*.
- 신규 저트래픽이라 커뮤니티가 한산 → "죽은 앱" 인상.

**전략**: 펫홈을 와꾸/톤으로 삼아 정체성·소유감·향수(싸이월드 세대)·바이럴(공유)을 얻고,
핵심 가치(AI펫톡·AI영상)는 펫홈 위의 *대표 액션*으로 올린다. 커뮤니티는 광장(1급)으로 유지.

## 1. 핵심 결정사항 (확정)

| # | 결정 |
|---|---|

| 10 | **3 surface 패리티 필수**: `src/`(데스크탑웹+모바일웹) + `mobile/`(앱) 동시 |

## 2. 명칭 체계 (화면 라벨) + 법적 회피

> 원칙: **메커니즘은 유지**(개인 공간·캐릭터·포인트·방명록·방문 = 보편 패턴, 보호대상 아님).
> **싸이월드 고유 명칭·스킨만 회피**. 코드 내부 식별자(minihompy/minimi 등)는 그대로 둬도 무방, **화면 라벨만 교체**.

| 현재(싸이월드색) | 새 화면 라벨 | 코드 내부명 | 비고 |
|---|---|---|---|
| 미니홈피 | **펫홈** | `minihompy` 유지 | 공간 = "우리 아이의 집" |
| 미니미 | **꼬미** | `minimi` 유지 | ✅ 확정 |
| 일촌 | **이웃** | `neighbor`(신규) | ✅ 인스타식: 단방향 팔로우 → 맞팔 시 양방향(=서로 이웃) |
| 도토리 | 포인트(그대로) | `points` | 이미 자체 명칭 |
| 방명록 | **방명록**(또는 발자국) | `guestbook` | 그대로 가능 |
| 파도타기 | **펫홈 탐험 / 마실가기** | — | 랜덤·오늘의 펫홈 탐색 |

> ※ 상표는 변호사 확인 권장. 실무적으로 "라벨 변경 + 자체 파스텔 디자인"이면 리스크 대부분 회피.

## 3. 정보구조 (IA) — 3 surface

네비 패러다임은 **그대로 둔다**(신규 도입 X). 바뀌는 건 *홈 화면 콘텐츠*와 *공개 펫홈 페이지*.

| | 네비 크롬 | 펫홈 위치 | 커뮤니티 |
|---|---|---|---|
| 데스크탑 웹 (`src/`) | 좌측 사이드바 | 기록 탭 내부 서브탭 | 메인 탭(1급) |
| 모바일 웹 (`src/` 반응형) | 상/하단 | 기록 안 | 메인 탭(1급) |
| 앱 (`mobile/`) | 하단 5탭: 기록·커뮤니티·**홈(중앙강조)**·AI펫톡·매거진 | 기록 안 (minihompy 탭 `href:null`) | 하단탭 2번(1급) |

- 세 surface 공통: **홈 = 펫홈 히어로 + 이웃/광장 피드**. 콘텐츠 동일, 크롬만 폼팩터별.
- 풀 펫홈(꾸미기·방명록) = 기록>펫홈(또는 히어로 탭).
- **공개 펫홈 = 독립 URL 페이지**(신규): 비회원도 보는 진짜 펫홈. → 4.4 / 9 참조.

## 4. 화면 설계

### 4.1 홈 — 비회원 (콜드 게스트)
목표: 3초 안에 "뭐 하는 곳 + 왜 지금" 전달 + 저마찰 첫 행동.
```
┌─────────────────────────────┐
│ [히어로]                     │
│  헤드라인: AI펫톡 + 펫홈     │  ← ✅ 둘을 어우러지게(차별점)
│   "우리 아이와 대화하고,     │     (기록/간직 추상카피 폐기)
│    펫홈에서 매일을 함께")    │     일러스트=AI대화 + 꾸민 펫홈 한 컷
│  일러스트(기존 재사용)       │
│  [카카오로 3초 시작] (주CTA) │  ← "시작하기" → 마찰제거 프레이밍
│  [둘러보기] (펫홈 탐험/커뮤) │
├─────────────────────────────┤
│ 지금 N명이 함께하고 있어요   │  ← 소셜프루프(visit_logs 재활용, C)
├─────────────────────────────┤
│ 오늘의 펫홈 (공개 펫홈 미리) │  ← 바이럴/구경거리 (탭→공개 펫홈)
│ 커뮤니티 인기글              │  ← 광장 노출(빈 앱 인상 완화)
└─────────────────────────────┘
```

### 4.2 홈 — 회원 (펫홈-forward + 광장 피드)
```
┌─────────────────────────────┐
│ [내 펫홈 히어로 카드]        │  ← 정체성. 배경+꼬미 배치+인사말(기존 미니홈피 프리뷰)
│  [우리 아이와 대화하기]      │  ← AI펫톡 (대표 액션)
│  [AI 영상 만들기]            │  ← AI영상 (대표 액션)
│  탭 → 풀 펫홈(꾸미기/방명록)  │
├─────────────────────────────┤
│ 이웃 소식                    │  ← 이웃의 새 글/펫홈 단장/추모 (river)
│ 오늘의 인기글 (커뮤니티)     │  ← 광장으로 끌어들임
│ 펫홈 탐험 (마실가기)         │  ← 랜덤/오늘의 펫홈
└─────────────────────────────┘
```
- **모바일 주의**: 첫 화면을 "순수 꾸민 방"만 띄우지 않는다. 펫홈은 *컴팩트 히어로 카드*, 본문은 피드 스크롤(고립 방지).

### 4.3 펫홈 — 본인 (풀뷰, 회원)
- 기존 `MiniHomepyTab`/`MinihompyStage` 재사용. 배경·꼬미 배치·인사말·방명록·방문수·좋아요수.
- 상단: [우리 아이와 대화하기] [AI 영상] [꾸미기] [공유].
- 일상/추모 토글은 펫 status(memorial)에 따름(§8).

### 4.4 펫홈 — 공개(타인/게스트 열람) ★바이럴 핵심
- **신규: 웹 독립 페이지**(현재 웹은 모달뿐). 예: `/pethome/[userId]` (URL 스킴 오픈이슈 C).
- 게스트가 링크로 도착 → 꾸며진 진짜 펫홈 + 인사말 + (있으면)대표 사진/추모.
- 보이는 것: 펫홈 공간, 방명록 *읽기*, 방문/좋아요 수, "이 아이와 대화해보기"(체험, 옵션 A).
- 상호작용 시도(방명록 쓰기·좋아요·이웃·대화 계속) → **가입 후크**(§7).
- 앱: 기존 `mobile/app/minihompy/[userId].tsx` 활용 + 게스트 대응 + 라벨.

### 4.5 커뮤니티 (광장) — 펫홈과 연결
- 1급 유지. 글 작성자 클릭 → 그 사람 **공개 펫홈**으로(이미 `router.push('/minihompy/[id]')` 앱 존재; 웹은 모달→페이지로).
- 펫홈 활동(단장/추모 전환)이 커뮤니티 "이웃 소식"으로 흘러나감.

## 7. 전환(가입) 후크 맵 + 카피(초안)

> 맨 모달 대신 **맥락 가치문구 + 소셜 3초**. 후크는 *소셜 액션*에 우선 배치(인텐트 높음).

| 트리거 | 카피(예) |
|---|---|
| 공개 펫홈에서 방명록 쓰기 | "○○에게 방명록을 남기려면 3초 가입 (카카오)" |
| 펫홈 좋아요/이웃 | "이웃하고 ○○ 소식 받기 — 카카오 3초" |
| AI펫톡 체험 후 | "계속 대화하려면 가입하고 우리 아이 펫톡 시작하기" |
| 펫홈 한참 구경(스크롤/시간) | (하단 소프트시트) "나도 우리 아이 펫홈 만들기 — 무료" |
| 커뮤니티 글쓰기/댓글/좋아요 | "이 글에 마음을 남기려면 3초 가입" |

## 8. 일상 ↔ 추모 표현 토글 규칙

- **데이터·공간 동일.** 추모 버튼(펫 status=memorial)에 따라 *표현만* 전환(기존 듀얼모드 활용).
- 게스트가 보는 공개 펫홈도 해당 펫 status를 따름.

| 요소 | 일상 모드 | 추모 모드 |
|---|---|---|
| 팔레트 | memento(하늘) | memorial(황금·은은) |
| 꼬미 동작 | 활발(터치 리액션) | 잔잔/정지 또는 숨김 |
| 주요 CTA | 꾸미기 / AI영상(fun) | 추억 보기 / 메시지 남기기 |
| 방명록 | 일상 방명록 | 추모 메시지(톤 카피) |
| AI펫톡 | 일상 케어 매니저 | 위로 상담 |
| 발랄 요소(배경효과 등) | 풀 | 절제 |

> ✅ 확정(단순화): **방명록·펫홈 공간은 그대로 공존**하고, 추모 모드에선 **UX/UI 톤앤매너만 전환**.
> 방명록을 별도 '추모 메시지' 기능으로 분리하지 않음 — 같은 방명록이 모드에 따라 분위기만 달라짐.
> 톤 원칙: "슬픔 게임화" 금지. 펫홈을 *'꾸미기/지르기'* 아닌 **'우리 아이의 공간·방문·메시지'** 로 프레이밍 (살아서 지은 집 → 떠난 뒤 찾아가는 공간 = 핵심 해자).

## 9. 공개 펫홈 · 공유 · 딥링크 (바이럴 루프)

- **획득 = 모바일 웹 공개 펫홈 / 깊이·리텐션 = 앱.**
- 공유 링크 → **모바일 웹 공개 펫홈**(설치 0). 앱 설치 유저는 딥링크(`mementoani://`)로 앱 펫홈.
- 현황: API public ✓ / 앱 라우트 ✓ / **웹 독립 페이지 ✗ (Phase 1 최우선 구축)**.
- 공유 버튼: 펫홈·게시글에 카카오 공유(기존 `KakaoShareButton`) 재사용, 미리보기(OG) = 펫홈 대표 이미지.

## 10. 데이터 / API 현황 → 필요분

| 영역 | 현황 | 필요 |
|---|---|---|
| 펫홈 조회 | `/api/minihompy/[userId]` GET **public** ✓ | 게스트 UX만 |
| 펫홈 설정 | `/api/minihompy/settings` ✓ | — |
| 방명록 | `/api/minihompy/[userId]/guestbook` ✓ | 게스트 쓰기 차단 후크 |
| 방문/좋아요 | `/visit` `/like` ✓ | — |
| 꼬미 | `/api/minimi/*` ✓ | 라벨만 |
| 방문자 통계 | `visit_logs` + `/api/admin/visits` ✓ (방금 구축) | "지금 N명" 공개 위젯용 집계 추가 |
| **이웃(neighbor)** | 없음 | **신규 테이블 + API** — 인스타식 단방향 팔로우(follow), 맞팔이면 양방향(=서로 이웃) |
| **공개 펫홈 웹페이지** | 없음(모달만) | **신규 `/pethome/[userId]` 페이지** |
| 이웃 소식 피드 | 없음 | 신규(이웃 활동 집계) |

## 11. 패리티 매핑 (구현 시 양쪽 동시)

| 기능 | 웹 `src/` | 앱 `mobile/` |
|---|---|---|
| 홈 | `app/page.tsx`, `components/features/home/*` (HeroSection 등) | `app/(tabs)/index.tsx`, `components/home/*` |
